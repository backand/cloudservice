Ionic 2 Kickstart

Follow these steps to install our Ionic 2 starter project, with Back& built-in!

Speed up your AngularJS development with super simple back-end and a complete and scalable gulpjs based build system that scaffolds the project for you.


Installation

Note: You can also find this kickstart guide on Github.
First, install the Ionic framework. This documentation is built against Ionic Framework version 2.0.0-rc.0. Once the Ionic framework is installed you can get the starter app using the following command:

    $ ionic start myApp https://github.com/backand/ionic2-starter1 --v2
    $ cd myApp

Then install dependencies for the project:

    $ npm i -S @backand/angular2-sdk socket.io-client @types/node @types/socket.io-client ionic-native

Next, install the Cordova plugin for Ionic.

    $ ionic plugin add cordova-plugin-inappbrowser

Finally, start the application:

    $ ionic serve

This brings up the starter app in a browser window. Log in to the starter app using a user configured in your application. At this point, you're done! Enjoy exploring the capabilities offered by our Ionic kickstart app. However, if you are interested in developing something more complex, read on for a quick overview of the functionality offered by this starter application.

Connecting to your own Backand application

If you want to customize the application's data model, or modify how authentication takes place, you'll need to create your own custom Backand application. To do so, follow these steps:
  1. Create a free application at Backand.com. This will replace the back end. This kickstart app is designed to work with the default data model without any issue.
  2. Change the parameters in the init function, found in /src/app/app.component.ts, to connect to your application: 
    
backand.init({
    appName: '{{vm.currentApp.Name || 'Your-App-Name'}}',
    SignUpToken: '{{vm.keys.signup || 'Your-SignUp-Token'}}',
    anonymousToken: '{{vm.keys.anonymous || 'Your-Anonymous-Token'}}',
    runSocket: true,
    mobilePlatform: 'ionic'
});

  3. Delete the starter content, and start writing your app!

CRUD functionality

This sample app features a demo of a CRUD service for an object called todo. You can see how we work through each of the basic REST methods for a CRUD object in the /src/pages/crud/crud.ts file. Expanding this to work with any object is a snap - simply replace todo with the name of the object in your Back& application that you wish to work with.

Authentication and Social Media Integration

The file /src/pages/signup/signup.ts demonstrates everything you need to create a new user in your application.
We also provide all the code you need to authenticate users with the service. Review the code in /src/pages/login/login.ts to see how we recommend you work with your user objects. You can use this code as a starter to build out the rest of your app's authentication scheme.

Installation

Note: You can also find this kickstart guide on the Ionic marketplace.
First, install the Ionic framework. This documentation is built against Ionic Framework version 2.0.0-rc.0. Once the Ionic framework is installed you can get the starter app using the following command:

    $ ionic start myApp https://github.com/backand/backand-ionic2-starter --v2
    $ cd myApp

Next, install the Cordova plugin for Ionic.

    $ ionic plugin add cordova-plugin-inappbrowser

Once Cordova is installed, update app.component.js lines 21-23 to use your Backand app's connection details

    backandService.setAppName('<your app name>');
    backandService.setSignUpToken('<your signup token>');
    backandService.setAnonymousToken('<your anonymousToken token>');

At this point, install the JS dependencies for the starter application:

    $ npm install socket.io-client --save
    $ npm install @types/node --save-dev --save-exact
    $ npm install @types/socket.io-client --save-dev --save-exact

Finally, start the application:

    $ ionic serve

This brings up the starter app in a browser window. Log in to the starter app using a user configured in your application. At this point, you're done! Enjoy exploring the capabilities offered by our Ionic kickstart app. However, if you are interested in developing something more complex, read on for a quick overview of connecting this starter to a custom Back& application.

Exploring the starter app

There is a lot of useful functionality buried in the starter app. Here are some highlights

CRUD functionality

This sample app features a demo of a CRUD service for an object called todo. You can see how we work through each of the basic REST methods for a CRUD object in the /src/pages/crud/crud.ts file. Expanding this to work with any object is a snap - simply replace todo with the name of the object in your Back& application that you wish to work with.

Authentication and Social Media Integration

The file /src/pages/signup/signup.ts demonstrates everything you need to create a new user in your application. It covers basic Back& users, as well as social media integration with providers like Google and Facebook. You can see how the signin methods work with social media providers by taking a look at the signup.ts file, line 39.
We also provide all the code you need to authenticate users with the service. Review the code in /src/pages/login/login.ts to see how we recommend you work with your user objects. You can use this code as a starter to build out the rest of your app's authentication scheme.

Facebook App Connect

Back& also gives you the capability to work with custom Facebook applications. To connect your Ionic2 Back& application to your Facebook app, you first need to install the Facebook-Cordova plugin for the Ionic framework:

    $ ionic plugin add cordova-plugin-facebook4 --save --variable APP_ID="" --variable APP_NAME=""

Once you've connected your application, you can use the inappSocial function of the BackandService module to interact with your app. You can see an example of this in file signup.ts.

Sockets

The Ionic 2 Starter also offers an example of our real-time communications service implemented with Socket.io. In the file crud.ts you can see an example of the application handling the items_updated event sent by the back-end. Here's the code in its entirety,, for reference:

    this.backandService.on("items_updated")
    .subscribe(
        data => {
            console.log("items_updated", data);
            let a = data as any[];
            let newItem = {};
            a.forEach((kv)=> newItem[kv.Key] = kv.Value);
            this.items.unshift(newItem);
        },
        err => {
            console.log(err);
        },
        () => console.log('received update from socket')
    );
